Documentation

Everything you need to know about using AFKmate.

Getting Started

Installation

  1. Open VS Code
  2. Go to Extensions (Cmd+Shift+X / Ctrl+Shift+X)
  3. Search for "AFKmate"
  4. Click Install
  5. Reload VS Code when prompted

First Analysis

AFKmate automatically starts monitoring when you open VS Code. Simply code as usual, and after 3 minutes of inactivity, AFKmate will analyze your modified files.

To trigger an immediate analysis, use the keyboard shortcut:

  • Mac: Cmd+Shift+A
  • Windows/Linux: Ctrl+Shift+A

Configuration

Settings

Open VS Code Settings and search for "AFKmate" to configure:

  • Idle Time: Minutes before analysis starts (default: 3)
  • Auto Start: Automatically analyze when idle (default: true)
  • Primary Provider: Choose between cloud or local Ollama
  • Notification Level: Control notification frequency
  • Show Errors/Warnings/Suggestions: Filter issue types

Using Ollama (Local LLM)

  1. Install Ollama from ollama.ai
  2. Pull a code model: ollama pull codellama
  3. Enable Ollama in AFKmate settings
  4. Set AFKmate's primary provider to "ollama"

Features

Auto-Fix

When AFKmate finds issues, it can generate fixes automatically:

  1. Click "Generate Fix" on any issue
  2. Preview the suggested changes
  3. Click "Apply Fix" to implement
  4. Use "Undo Fix" if needed

Export Results

Export analysis results for documentation or sharing:

  • Use AFKmate: Export Results command
  • Choose JSON or Markdown format
  • Copy to clipboard with AFKmate: Copy Results

Keyboard Shortcuts

  • Cmd/Ctrl+Shift+A: Analyze Now
  • Cmd/Ctrl+Shift+P: Pause/Resume Analysis
  • Cmd/Ctrl+Shift+U: Check Usage
  • Cmd/Ctrl+Shift+F: Fix All Issues
  • Cmd/Ctrl+Shift+Z: Undo Last Fix

Troubleshooting

Analysis Not Starting

  • Check if analysis is paused (status bar indicator)
  • Verify you have remaining free analyses
  • Ensure you're connected to the internet (or using Ollama)

Slow Analysis

  • Large files take longer to analyze
  • Check your internet connection
  • Consider using local Ollama for faster results
